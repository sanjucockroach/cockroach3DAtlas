    95→## 5. Product Vision — The UPSC World Atlas
    96→
    97→### 5.1 What it is
    98→A full-screen **3D WebGL globe** of the Earth showing **all countries with demarcated boundaries**, overlaid with **layers of UPSC-relevant hotspots**. Clicking any hotspot opens a **branded detail sidebar**. The globe is filterable by syllabus lens, searchable, and bookmarkable.
    99→
   100→### 5.2 The layer taxonomy (the “many more” you mentioned, made concrete)
   101→
   102→| # | Layer | Source | UPSC relevance |
   103→|---|---|---|---|
   104→| L1 | **Live Earth Events** | NASA EONET v3 (`/events/geojson`) | Disaster management, GS3, geography |
   105→| L2 | **Volcanoes & Seismic** | EONET `category=volcanoes` + curated | Geomorphology GS1 |
   106→| L3 | **Places in News** | Curated (+ scraping-ready) | GS2 IR, current affairs |
   107→| L4 | **Biodiversity Hotspots** | Curated (36 global hotspots) | GS3 environment |
   108→| L5 | **Ramsar Wetlands** | Curated (India + global) | GS3 environment |
   109→| L6 | **UNESCO Heritage Sites** | Curated | GS1 culture, GS3 |
   110→| L7 | **Straits · Canals · Passes** | Curated | GS1 geo, GS2 IR |
   111→| L8 | **Rivers · Mountains · Deserts** | Curated | GS1 geo |
   112→| L9 | **Conflict & Border Zones** | Curated | GS2 IR, internal security |
   113→| L10 | **Treaties / Org HQ** | Curated | GS2 IR |
   114→| L11 | **Mineral & Industrial Regions** | Curated | GS3 economy |
   115→| L12 | **Trade / Migration Arcs** | Curated arcs on globe | GS2/GS3 |
   116→
   117→Every hotspot carries **syllabus tags** + **PYQ links** + **mains hooks** (see §13).
   118→
   119→### 5.3 Why this is “no one can reach”
   120→- Most UPSC map tools are **static PDF atlases** or 2D clickable maps. We give a **live, 3D, NASA-fed, syllabus-mapped, PYQ-linked** instrument that *updates itself* with real Earth events. That combination is genuinely rare.

   157→## 7. Data Architecture
   158→
   159→### 7.1 Two data families, one globe
   160→```
   161→                ┌───────────────────────────────────────────────┐
   162→                │              UPSC WORLD ATLAS DATA             │
   163→                └───────────────────────────────────────────────┘
   164→                                 │
   165→          ┌──────────────────────┴──────────────────────┐
   166→          ▼                                             ▼
   167→  ┌───────────────────┐                     ┌────────────────────┐
   168→  │  LIVE (dynamic)   │                     │  CURATED (static)  │
   169→  │  NASA EONET v3    │                     │  UPSC knowledge    │
   170→  └───────────────────┘                     └────────────────────┘
   171→          │                                             │
   172→   server fetch+cache                           Prisma (SQLite) +
   173→   (Next API route)                             seed JSON file
   174→          │                                             │
   175→          └──────────────────┬──────────────────────────┘
   176→                             ▼
   177→                 unified GeoJSON feed to the globe
   178→```
   179→
   180→### 7.2 Live layer — NASA EONET
   181→- **Endpoints used:** `/events/geojson`, `/categories`, `/sources`, `/layers`, `/magnitudes`.
   182→- **Proxy via Next.js API route** (`/api/atlas/eonet`) to: avoid browser CORS, cache server-side, normalise to a unified hotspot shape, attach syllabus tags by category.
   183→- **Cache:** in-memory cache, **TTL 20 min** (EONET updates roughly hourly; 20 min balances freshness vs. rate limits). Stale-while-revalidate.
   184→- **Filters supported:** `status` (open/all), `category`, `days`, `limit`, `bbox` (for region deep-dive).
   185→- **Category → syllabus mapping** (auto-tag):
   186→  | EONET category | UPSC tag |
   187→  |---|---|
   188→  | wildfires | GS3 environment / disaster mgmt |
   189→  | severeStorms | GS1 climatology / GS3 disaster |
   190→  | volcanoes | GS1 geomorphology |
   191→  | earthquakes | GS1 geomorphology / GS3 disaster |
   192→  | floods | GS3 disaster mgmt |
   193→  | landslides | GS3 disaster mgmt |
   194→  | seaLakeIce | GS1 climatology |
   195→  | snow | GS1 climatology |
   196→  | dustHaze | GS1 environment |
   197→  | manMade | GS3 internal security / economy |
   198→  | tempExtremes | GS3 climate change |
   199→
   200→### 7.3 Curated UPSC layer
   201→- Stored in **Prisma (SQLite)** so it is editable later (admin route in v2) and survives restarts.
   202→- **Schema sketch** (final in build):
   203→  ```
   204→  model Hotspot {
   205→    id          String   @id @default(cuid())
   206→    layer       String   // L3..L11
   207→    title       String
   208→    description String
   209→    lat         Float
   210→    lng         Float
   211→    country     String?
   212→    region      String?
   213→    syllabusTags String  // "GS1:Geo,GS2:IR"
   214→    pyqIds      String?  // comma-sep refs
   215→    mainsHooks  String?  // newline-sep
   216→    sources     String?  // JSON array of {title,url}
   217→    image       String?
   218→    createdAt   DateTime @default(now())
   219→    updatedAt   DateTime @updatedAt
   220→  }
   221→  model Pyq {
   222→    id       String @id @default(cuid())
   223→    year     Int
   224→    paper    String // "Prelims" / "Mains GS1"
   225→    question String
   226→    answer   String?
   227→    hotspotIds String? // back-link
   228→  }
   229→  ```
   230→- **Seed file:** `prisma/seed-atlas.ts` bootstraps ~120–180 curated hotspots across L3–L11 to start (I author these, you review).
   231→- **Refresh:** curated layer is versioned; live EONET refreshes itself.
   232→
   233→### 7.4 Unified hotspot shape (what the globe renders)
   234→```ts
   235→type AtlasHotspot = {
   236→  id: string
   237→  source: 'eonet' | 'curated'
   238→  layer: string                 // L1..L12
   239→  category: string              // eonet category or curated type
   240→  title: string
   241→  description?: string
   242→  lat: number
   243→  lng: number
   244→  geometry?: GeoJSON.Geometry   // polygons for EONET area events
   245→  magnitude?: { value: number; unit: string }
   246→  date?: string                 // ISO
   247→  status?: 'open' | 'closed'
   248→  syllabusTags: string[]
   249→  pyqs?: PyqRef[]
   250→  mainsHooks?: string[]
   251→  sources?: { title: string; url: string }[]
   252→}
   253→```

   259→### 8.1 Stack (all already in your project — no new heavy deps except the globe lib)

   315→### 8.5 Performance & resilience
   316→- EONET fetched **server-side only**, cached 20 min, never from the browser.
   317→- Country GeoJSON served **locally** (bundled in `/public`), so boundaries always render even if external CDNs fail.
   318→- Globe points **clustered** above a density threshold to keep 60fps.
   319→- `prefers-reduced-motion` respected (fly-to becomes instant cut).
   320→- Graceful fallback: if EONET is down, show curated layers + a small “live events temporarily unavailable” pill (no broken map).

   621→## 13. Syllabus Mapping Logic (the brains)
   622→
   623→Every hotspot, live or curated, resolves to a tag set:
   624→```
   625→syllabusTags = [ paper, subject, subtopic? ]
   626→e.g. ["GS2","IR","Energy security"]
   627→     ["GS3","Disaster management","Floods"]
   628→     ["GS1","Geomorphology","Volcanism"]
   629→```
   630→- **EONET auto-tags** via category→tag map (§7.2).
   631→- **Curated hotspots** hand-tagged at seed time.
   632→- **PYQ link** = `Hotspot.pyqIds` → renders year + paper + question + answer.
   633→- **Mains hooks** = 3–5 bullet dimensions (causes, implications, India’s stakes, way forward) authored per curated hotspot; for EONET live events, generated from a template until Tier-3 LLM enrichment.
   634→
   635→This is what makes the Atlas a *cognitive instrument*, not a map widget.
